留学生管理平台｜技术方案

发表于 2016-09-22   |   分类于 计划   |     |  

技术方案

系统架构

B/S和C/S，整个完整的系统包括移动端学生教师管理app，web前台界面，后台管理界面。考虑实际的业务需求，需要分两个业务模块。

后台框架

  • 后台采用Spring＋SpringMVC＋Mybatis＋activiti等框架，便于进行报名流程的管理。该平台的技术栈如下，留学生信息管理平台技术栈
    • 使用目前主流的Java EE开发框架，简单易学，学习成本低。
    • 数据库无限制，目前支持MySql、Oracle，可扩充SQL Server、PostgreSQL、H2等。
    • 模块化设计，层次结构清晰。内置一系列企业信息管理的基础功能。
    • 操作权限控制精密细致，对所有管理链接都进行权限验证，可控制到按钮。
    • 数据权限控制精密细致，对指定数据集权限进行过滤，七种数据权限可供选择。
    • 提供在线功能代码生成工具，提高开发效率及质量。
    • 提供常用工具类封装，日志、缓存、验证、字典、组织机构等，常用标签（taglib），获取当前组织机构、字典等数据。
    • 兼容目前最流行浏览器（IE7+、Chrome、Firefox）IE6也支持，但体验效果差。
  • activiti 5.0(Java)
    • 可扩展性/性能/可靠性
    • 更有利于多人协作
    • 向下兼容
    • 大数据的发展趋势，java值得深入学习，java的性能更好

数据库

  • mysql数据库,后期考虑扩充PostgreSQL
    • mysql基于线程，运行速度快，但数据类型不够丰富
    • PostgreSQL面向对象，数据类型丰富，支持json等。

前端框架

  • bootstrap v3 http://bootcss.com

    • 最成熟和最广泛使用的Html/css框架，支持IE8+，提供各类常见布局和组件
    • 社区支持良好，尤其适合快速界面开发和后台界面开发
    • 对于学习前端知识，可提供一个不错的结构性认识

  • jquery 1.9

    • 最广泛使用的javascript基础库
      应有尽有插件支持
    • 虽然从当前的前端发展方向来看，上述技术均有些落后，但对于普通的WEB开发，这些技术已经足够。

移动端

对比appcan等框架选定，看中混合开发特点。

  • react native
    • 先进的逻辑层开发框架
    • 灵活的 UI 开发框架。
    • 使用JavaScriptCore进行应用的升级。

开发环境

  • mac os
  • sublime text 3及各类插件
    • ReactJS : 支持React开发，代码提示，高亮显示
    • Emmet ：前端开发必备。
    • Terminal : 在sublime中打开终端并定位到当前目录，神器，mac下的快捷键为：command+shift+T
    • react-native-snippets：react native 的代码片段
  • 最新chrome dev 版本
  • git

部署环境

  • OS： CentOS 7 server
  • nginx : 负载均衡、反向代理
  • tomcat：WEB服务器
